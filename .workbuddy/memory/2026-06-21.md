## TradeLeadAgent 搜索质量全面改进

### 问题反馈
1. 搜索样本只有20-40条，显著不够
2. 结果含大量平台/国内/垃圾数据，抓取电话号码大部分是垃圾
3. 希望提升数据范围、精确度，确认是有意向的客户商

### 修复内容

#### 1. 提升搜索样本量（searcher.py + app.py）
- `max_results` 滑块：10-50 → **10-200**，默认 **50**
- `search_web` 每 query 最少返回 **10 条**（之前是 `max_results // len(queries) + 2`，默认 20→ 仅 6 条/query）
- 扩展搜索 query 数量：5个 → **12个**（含海关数据/展会/exhibitor 关键词）
- 新增多语言 query（西班牙语/葡萄牙语/法语/德语）

#### 2. 优化过滤逻辑（searcher.py）
- **放宽 `_is_relevant`**：移除强制关键词要求（之前 `REQUIRED_KEYWORDS` 导致大量误杀）
- 新增 `GOOD_DOMAINS` 白名单（Kompass/Europages/ThomasNet/ImportGenius等），白名单域名直接保留
- 新增 `JUNK_SIGNALS` 垃圾信号（招聘/登录/新闻），仅排除明显无意义页面
- 扩展 `EXCLUDED_DOMAINS`：新增 `.cn` 域名排除（目标市场非中文区时）

#### 3. 新增海关数据/展会数据源（searcher.py）
- 新增 `search_customs_and_tradeshow()` 函数
- 新增 `_build_customs_queries()` 和 `_build_tradeshow_queries()`
- 搜索关键词：`"bill of lading"`, `"customs data"`, `"trade show exhibitor"`, `"exhibition exhibitor list"`
- `app.py` 新增数据源选项「海关数据/展会」（默认开启）

#### 4. 改进邮箱/电话抓取质量（scraper.py）
- **邮箱**：优先 `info@/contact@/sales@/hello@` 等联系类邮箱；过滤 `support@/admin@/abuse@` 等非采购邮箱
- **电话**：严格验证长度（7-15位）；过滤全同数字号、全零号；最多返回 8 个
- 扩展 About/Contact 页面抓取：最多尝试 3 个链接（之前仅 2 个）

#### 5. app.py 更新
- 导入 `search_customs_and_tradeshow`
- 数据源多选默认：`["Web搜索", "商业目录", "海关数据/展会"]`（LinkedIn/社交媒体不再默认勾选）
- 各数据源 `max_results` 传参提升：商业目录 now gets `max_results`（之前是 `//2`）


### 6. 修复 MARKET_KEY_WORDS 命名不一致导致的 AttributeError（第2次修复）
**根因**：`config.py` 定义的是 `MARKET_KEYWORDS`（无下划线），但 `searcher.py` 里查找的是 `MARKET_KEY_WORDS`（有下划线）。`getattr` 返回空 `{}` 后 fallback 到 `target_market` 本身，但某些路径下触发异常。
**修复**：
- `config.py` 新增 `MARKET_KEY_WORDS = MARKET_KEYWORDS` 别名
- `searcher.py` 6 处 `getattr` 全部改为 `getattr(config, "MARKET_KEY_WORDS", None) or getattr(config, "MARKET_KEYWORDS", {})`，同时兼容两种命名

## 自修复 Agent Loop 体系设计完成

### 背景
用户要求设计一个 agent loop，能持续自我检查、修复、迭代 TradeLeadAgent 网页客户端，不停运行直到手动停止。

### 体系架构（双层）

#### Layer 1：云端自动化（Automation）
- 任务名：`TradeLeadAgent Self-Heal`
- 频率：每小时执行一次
- 动作：
  1. 运行 `scripts/health_check.py` 生成诊断报告
  2. 如果发现问题，运行 `scripts/auto_fix.py` 自动修复
  3. 修复后等待 30 秒，再次运行 health_check 验证
  4. 最多尝试 3 次，成功后自动 `git commit && git push`
  5. 记录修复日志到 `.workbuddy/memory/auto-heal-{date}.md`

#### Layer 2：本地持续监控（self_heal_loop.py）
- 脚本：`python scripts/self_heal_loop.py`
- 频率：每 5 分钟检查一次，发现问题后每 1 分钟重试修复
- 终止：按 `Ctrl+C` 随时停止
- 与云端独立运行，适合本地开发阶段密集监控

### 脚本说明

| 脚本 | 功能 | 输出 |
|------|------|------|
| `health_check.py` | 6 维度诊断：语法检查、模块导入、config/searcher 兼容性、app 导入一致性、函数签名、scraper 配置 | JSON 报告 + 退出码 |
| `auto_fix.py` | 5 类自动修复：market_keywords 别名、searcher 引用安全化、scraper 引用安全化、app 导入 stub、缺失 init 导出 | JSON 报告 + 是否有变更 |
| `self_heal_loop.py` | 本地无限循环：检查→修复→提交→验证→日志 | 控制台实时输出 + 日志文件 |
